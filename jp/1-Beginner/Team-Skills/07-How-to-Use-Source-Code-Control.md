# How to Use Source Code Control
[//]: # (Version:1.0.0)
ソースコード管理システム（バージョン管理システムとも呼ばれます）では、プロジェクトを効果的に管理できます。彼らは一人にとってとても有用で、グループには必要不可欠です。コードが失われず、変更に意味を割り当てることができるように、異なるバージョンのすべての変更を追跡します。あなたが変更するコードは、チームと共有される、またはリリースされるコミットされた公式のコードから慎重に保たれるので、ソースコード管理システムに自信を持ってスローアウェイとデバッグコードを作成することができます。

私はソースコード管理システムの利点を理解するのに遅れましたが、今は1人のプロジェクトでもなくても生きていません。一般的には、同じコードベースで作業するチームが必要な場合に必要です。しかし、彼らには別の大きな利点があります。コードを成長しているオーガニックシステムと考えることを奨励します。各変更は新しい名前または番号の新しいリビジョンとしてマークされるため、ソフトウェアは目に見える進歩的な一連の改善と考えるようになります。私はこれが初心者にとって特に便利だと思います。

ソースコード管理システムを使用するための良いテクニックは、常に最新の状態になってから数日以内に滞在することです。数日で終了できないコードはチェックインされていますが、非アクティブで呼び出されることはないため、他の人には問題は生じません。あなたのチームメイトを減速させる間違いを犯すことは重大なエラーです。それはしばしばタブーです。
